10  Filas de prioridade: o heap binário

A fila do capítulo 8 atende por ordem de chegada (FIFO). Mas muitos problemas pedem outra coisa: atender por prioridade — o mais urgente primeiro, não importa quando chegou. É a fila de prioridade, e sua implementação eficiente é uma das estruturas mais elegantes do livro: o heap binário. Ele mantém o elemento de maior prioridade sempre à mão e suporta inserir e remover em \(O(\log n)\) — e, surpresa, vive num array comum, sem um único ponteiro.

10.1 A analogia: a triagem do pronto-socorro

Num pronto-socorro, não se atende por ordem de chegada — atende-se por gravidade. Quem chega infartando passa na frente de quem espera há uma hora com um corte no dedo. A cada chamada, a equipe precisa achar o caso mais grave rapidamente, e quando um novo paciente chega, encaixá-lo na ordem de urgência sem reorganizar a sala inteira.

É exatamente o contrato de uma fila de prioridade: achar o mínimo (o mais urgente) e inserir novos elementos, ambos eficientes. Poderíamos manter tudo ordenado num array — aí o mínimo é o primeiro, \(O(1)\). Mas inserir exigiria abrir espaço no meio, \(O(n)\). O heap encontra o equilíbrio: nem busca do mínimo nem inserção custam mais que \(O(\log n)\).

10.2 A árvore que é, na verdade, um array

O heap é uma árvore binária completa: todos os níveis cheios, exceto talvez o último, preenchido da esquerda para a direita. Essa regularidade permite um truque lindo — guardar a árvore num array, sem ponteiros, mapeando posições por aritmética de índice:

\[ \text{filho esq.}(i) = 2i+1 \qquad \text{filho dir.}(i) = 2i+2 \qquad \text{pai}(i) = \left\lfloor \tfrac{i-1}{2} \right\rfloor \]

Ler o array da esquerda para a direita é percorrer a árvore nível a nível (BFS). A Figura 10.1 mostra as duas faces da mesma estrutura:

Guardar a árvore num array dá ao heap o melhor dos dois mundos: a navegação hierárquica de uma árvore (subir e descer por índice) com a memória contígua e a localidade de cache de um array (capítulo 5) — e zero ponteiros, zero malloc por nó. A estrutura de árvore é puramente conceitual; fisicamente, é só um vetor crescível.

10.3 A invariante do heap

A regra de um min-heap é deliberadamente mais frouxa que a da BST:

Todo pai é menor ou igual a seus filhos.

Só isso. Não há ordem entre irmãos, nem entre esquerda e direita — apenas a relação vertical pai/filho. A consequência é poderosa: o menor elemento de todos está sempre na raiz, em [0], acessível em \(O(1)\). Essa invariante mais fraca é mais barata de manter que a da BST, e é tudo o que uma fila de prioridade precisa.

10.4 Inserir e subir: sift-up

Para inserir, colocamos o novo elemento no fim do array (a próxima folha) e o deixamos subir trocando com o pai enquanto for menor — restaurando a invariante ao longo de um único caminho até a raiz:

// Sobe o elemento i enquanto for menor que o pai. O(log n).
static void sift_up(MinHeap *h, size_t i) {
    while (i > 0) {
        size_t parent = (i - 1) / 2;
        if (h->data[i] >= h->data[parent]) {
            break; // o pai já é menor-ou-igual: posição correta
        }
        swap(&h->data[i], &h->data[parent]);
        i = parent;
    }
}
// Insere no fim do array e "sobe" o novo elemento até seu lugar.
bool heap_push(MinHeap *h, int value) {
    if (!ensure_cap(h, h->size + 1)) {
        return false;
    }
    h->data[h->size] = value;
    h->size++;
    sift_up(h, h->size - 1);
    return true;
}

Como a árvore é completa, sua altura é \(\lfloor \log_2 n \rfloor\), e sift-up percorre no máximo essa altura: inserção em \(O(\log n)\).

10.5 Remover o mínimo e descer: sift-down

Extrair o mínimo é o espelho: o resultado é a raiz ([0]). Para preencher o buraco sem quebrar a forma completa, trazemos o último elemento para a raiz e o deixamos descer, sempre trocando com o menor dos filhos, até obedecer à invariante:

// Desce o elemento i, trocando com o MENOR filho, até obedecer à regra.
static void sift_down(MinHeap *h, size_t i) {
    for (;;) {
        size_t left = 2 * i + 1;
        size_t right = 2 * i + 2;
        size_t smallest = i;
        if (left < h->size && h->data[left] < h->data[smallest]) {
            smallest = left;
        }
        if (right < h->size && h->data[right] < h->data[smallest]) {
            smallest = right;
        }
        if (smallest == i) {
            break; // já é menor que ambos os filhos
        }
        swap(&h->data[i], &h->data[smallest]);
        i = smallest;
    }
}
// Remove e devolve o MÍNIMO (a raiz).
bool heap_pop(MinHeap *h, int *out) {
    if (h->size == 0) {
        return false;
    }
    *out = h->data[0];                 // o mínimo está sempre na raiz
    h->data[0] = h->data[h->size - 1]; // o último elemento vai para a raiz
    h->size--;
    if (h->size > 0) {
        sift_down(h, 0); // e desce até o lugar certo
    }
    return true;
}

Também \(O(\log n)\) — um único caminho da raiz até, no máximo, uma folha.

10.6 Montar de uma vez: build-heap em \(O(n)\)

Se você já tem todos os elementos num array, dá para transformá-lo num heap mais rápido do que inserir um a um. O algoritmo de Floyd aplica sift-down de baixo para cima, começando no último nó interno:

// Monta um heap a partir de um array qualquer em O(n) (algoritmo de Floyd):
// heapifica de baixo para cima, do último nó interno (n/2 - 1) até a raiz.
bool heap_build(MinHeap *h, const int *values, size_t n) {
    if (!ensure_cap(h, n)) {
        return false;
    }
    for (size_t i = 0; i < n; i++) {
        h->data[i] = values[i];
    }
    h->size = n;
    for (size_t i = n / 2; i-- > 0;) {
        sift_down(h, i);
    }
    return true;
}
DicaComplexidade (Big-O)

Inserir \(n\) elementos um a um custa \(O(n \log n)\). Mas o build-heap de Floyd faz o mesmo em \(O(n)\) — e a razão é elegante: a maioria dos nós está perto do fundo e quase não desce (sift-down curto), enquanto os poucos nós altos, que descem muito, são raros. A soma \(\sum_{h} \frac{n}{2^{h+1}} \cdot h\) converge para \(O(n)\), não \(O(n \log n)\). Quando puder, construa de uma vez em vez de inserir em laço.

10.7 Heapsort de brinde

Há um bônus escondido. Se você insere tudo num min-heap e depois extrai o mínimo repetidamente, os elementos saem em ordem crescente — porque cada pop devolve o menor que restou. Isso é o heapsort: ordenar em \(O(n \log n)\) usando só as operações do heap, sem memória auxiliar além do próprio array.

10.8 Vendo funcionar

O programa insere sete valores, espia o mínimo, esvazia o heap (heapsort) e por fim monta outro heap com build-heap:

int main(void) {
    int xs[] = {5, 3, 8, 1, 9, 2, 7};
    size_t n = sizeof(xs) / sizeof(xs[0]);

    // Inserindo um a um; o mínimo borbulha para a raiz.
    MinHeap h;
    heap_init(&h);
    for (size_t i = 0; i < n; i++) {
        heap_push(&h, xs[i]);
    }
    int m;
    heap_peek(&h, &m);
    printf("min (peek): %d\n", m);

    // Extrair tudo, repetidamente, sai em ordem CRESCENTE — isto é o heapsort.
    printf("extraindo (heapsort):");
    int v;
    while (heap_pop(&h, &v)) {
        printf(" %d", v);
    }
    printf("\n");
    heap_free(&h);

    // build-heap: monta o heap a partir do array de uma vez, em O(n).
    MinHeap h2;
    heap_init(&h2);
    heap_build(&h2, xs, n);
    heap_peek(&h2, &m);
    printf("build-heap O(n), min: %d\n", m);
    heap_free(&h2);

    return 0;
}

10.9 Pense como projetista

DicaPense como projetista

Use um heap quando precisar repetidamente do extremo (mínimo ou máximo) de um conjunto que muda — e não da ordenação completa. Se você só quer “o próximo mais urgente”, o heap dá isso em \(O(\log n)\) sem o custo de manter tudo ordenado. Mas se precisa buscar um elemento arbitrário (não o mínimo), o heap não ajuda: ele não tem a ordem esquerda-direita da BST, então procurar um valor no meio é \(O(n)\). Heap é a ferramenta do “qual é o próximo?”, não do “onde está o X?”.

NotaNo mundo real

A fila de prioridade é o motor de algoritmos de caminho mínimo: o Dijkstra e o A* (usado em GPS e em jogos) escolhem sempre o próximo nó mais promissor extraindo o mínimo de um heap. Escalonadores de tarefas, compressão (o Huffman do próximo capítulo!), simulação de eventos discretos e o heapsort também vivem disso. Para grafos enormes, há variantes que melhoram a teoria — como a Fibonacci heap, que torna certas operações \(O(1)\) amortizado, ao custo de uma implementação bem mais intrincada.

10.10 Resumo

  • Uma fila de prioridade atende pelo elemento de maior prioridade (o mínimo), não por ordem de chegada. O heap binário a implementa com inserção e remoção em \(O(\log n)\) e acesso ao mínimo em \(O(1)\).
  • O heap é uma árvore binária completa guardada num array: filhos de i em 2i+1 e 2i+2, pai em (i-1)/2. Sem ponteiros, ótima localidade de cache.
  • A invariante (pai \(\le\) filhos) é mantida por sift-up (na inserção) e sift-down (na remoção), cada um percorrendo uma única altura \(O(\log n)\).
  • build-heap monta um heap a partir de um array em \(O(n)\) — melhor que inserir um a um. E extrair tudo dá o heapsort, \(O(n \log n)\).
