11  O desafio final: compactando com Huffman

Chegamos ao topo. Este capítulo não apresenta uma estrutura nova — ele junta quase todas. Vamos construir um compactador de arquivos de verdade, com o algoritmo de Huffman, e no caminho usar o histograma (arrays), o heap do capítulo 10, uma árvore binária, recursão para gerar códigos e a manipulação de bits que vem lá dos fundamentos. É o livro inteiro trabalhando junto para resolver um problema real: gastar menos espaço.

A ideia central é desconfiar do óbvio. Um arquivo de texto gasta 8 bits fixos para cada byte — o 'a' e o 'z' custam o mesmo, embora o 'a' apareça muito mais. E se déssemos códigos curtos aos bytes frequentes e longos aos raros? Em média, o arquivo encolhe. Isso é Huffman.

11.1 A analogia: o código Morse

O código Morse já fazia isso em 1840. As letras mais comuns do inglês recebem os sinais mais curtos: E é um único ponto (·), T um único traço (–). As raras ficam longas: Q é – – · –. Quem projetou o Morse contou (na mão) a frequência das letras e deu códigos curtos às campeãs. Huffman faz exatamente isso, mas de forma automática e comprovadamente ótima — nenhum outro código de tamanho variável que respeite as frequências bate o de Huffman.

11.2 O problema dos prefixos

Códigos de tamanho variável têm uma armadilha. Se a = 0 e b = 01, ao ler 0... o decodificador não sabe se já viu um a ou se é o começo de um b. A solução é usar um código livre de prefixo: nenhum código pode ser o começo de outro. Aí a leitura é sempre sem ambiguidade.

E como garantir isso? Com uma árvore. Se cada símbolo mora numa folha, e o código é o caminho da raiz até ela (0 = esquerda, 1 = direita), então nenhum código é prefixo de outro — porque nenhuma folha está no caminho de outra. A árvore é o código. Todo o algoritmo gira em torno de construir a melhor árvore possível.

11.3 Etapa (a): o histograma

Tudo começa contando. Percorremos os dados uma vez e contamos quantas vezes cada um dos 256 valores de byte aparece — um array de 256 contadores:

// (a) Histograma: conta quantas vezes cada um dos 256 bytes aparece.
static void count_freq(const unsigned char *data, size_t n, size_t freq[256]) {
    for (int i = 0; i < 256; i++) {
        freq[i] = 0;
    }
    for (size_t i = 0; i < n; i++) {
        freq[data[i]]++;
    }
}

11.4 Etapa (b): a árvore de Huffman

Agora a sacada genial. Pegamos os dois símbolos menos frequentes e os juntamos sob um nó-pai, cuja frequência é a soma dos dois. Esse pai volta para o monte, e repetimos: sempre unindo os dois nós de menor frequência. Os raros, unidos cedo, afundam para longe da raiz (códigos longos); os frequentes ficam rasos (códigos curtos). “Retirar sempre os dois menores” é, literalmente, dois pop da fila de prioridade — o heap do capítulo 10, agora guardando nós de árvore:

// (b) Árvore de Huffman: enquanto houver mais de um nó, retire os DOIS de menor
// frequência (o heap entrega isso) e junte-os sob um pai cuja frequência é a
// soma. O último nó que sobra é a raiz. Os mais frequentes ficam mais rasos.
static HuffNode *build_tree(const size_t freq[256]) {
    NodeHeap heap = {0};
    for (int b = 0; b < 256; b++) {
        if (freq[b] > 0) {
            nh_push(&heap, leaf_new((unsigned char)b, freq[b]));
        }
    }
    if (heap.size == 0) {
        free(heap.data);
        return NULL; // entrada vazia
    }
    if (heap.size == 1) {
        // Um único símbolo distinto: garante um código de 1 bit com uma folha
        // "fantasma" (que nunca é codificada, mas dá forma à árvore).
        HuffNode *only = nh_pop(&heap);
        nh_push(&heap, only);
        nh_push(&heap, leaf_new(only->byte, 0));
    }
    while (heap.size > 1) {
        HuffNode *a = nh_pop(&heap);
        HuffNode *b = nh_pop(&heap);
        nh_push(&heap, internal_new(a, b));
    }
    HuffNode *root = nh_pop(&heap);
    free(heap.data);
    return root;
}

A Figura 11.2 mostra a árvore para a palavra abracadabra (a:5, b:2, r:2, c:1, d:1). Siga os caminhos: o a, frequentíssimo, fica a um passo da raiz (código 0); os raros c e d ficam a três passos (100, 101).

11.5 Etapa (c): o dicionário de códigos

Com a árvore pronta, geramos a tabela de códigos descendo recursivamente: a cada passo à esquerda anexamos um 0, à direita um 1, e ao chegar numa folha registramos o código acumulado para aquele byte.

// (c) Dicionário: desce a árvore acumulando bits (0 = esquerda, 1 = direita).
// Cada folha recebe seu código — guardado como os `len` bits baixos de `bits`.
typedef struct {
    uint32_t bits;
    uint8_t len;
} Code;

static void gen_codes(const HuffNode *node, uint32_t bits, uint8_t len,
                      Code table[256]) {
    if (node->left == NULL && node->right == NULL) { // folha
        table[node->byte].bits = bits;
        table[node->byte].len = (len == 0) ? 1 : len; // raiz-folha → 1 bit
        return;
    }
    gen_codes(node->left, (bits << 1) | 0u, (uint8_t)(len + 1), table);
    gen_codes(node->right, (bits << 1) | 1u, (uint8_t)(len + 1), table);
}

11.6 Etapa (d): a manipulação de bits

Aqui está a parte que mais assusta e mais ensina. Os códigos têm tamanhos quaisquer — 1 bit, 3 bits, 7 bits — mas a memória só endereça bytes (8 bits). Precisamos juntar bits soltos num acumulador e, a cada 8, cuspir um byte — e fazer o inverso na leitura. É a essência de qualquer formato binário:

// Escreve UM bit. Acumula 8 bits e só então grava um byte (MSB primeiro).
static bool bw_put_bit(BitWriter *w, int bit) {
    w->acc = (w->acc << 1) | (unsigned)(bit & 1);
    w->nbits++;
    if (w->nbits == 8) {
        if (!bw_reserve(w, w->size + 1)) {
            return false;
        }
        w->bytes[w->size++] = (unsigned char)w->acc;
        w->acc = 0;
        w->nbits = 0;
    }
    return true;
}

// No fim, grava o byte parcial preenchendo o resto com zeros (padding).
static bool bw_flush(BitWriter *w) {
    if (w->nbits > 0) {
        w->acc <<= (8 - w->nbits);
        if (!bw_reserve(w, w->size + 1)) {
            return false;
        }
        w->bytes[w->size++] = (unsigned char)w->acc;
        w->acc = 0;
        w->nbits = 0;
    }
    return true;
}

A leitura é o espelho: um deslocamento e uma máscara extraem o bit na posição certa do byte atual, devolvendo -1 quando os dados terminam.

// Lê UM bit (MSB primeiro). Devolve 0/1, ou -1 quando os dados acabam.
static int br_get_bit(BitReader *r) {
    if (r->pos >= r->size) {
        return -1;
    }
    int bit = (r->bytes[r->pos] >> (7 - r->nbits)) & 1;
    r->nbits++;
    if (r->nbits == 8) {
        r->pos++;
        r->nbits = 0;
    }
    return bit;
}

11.7 Etapa (e): juntando tudo

Falta um detalhe crucial: o descompactador precisa da mesma árvore para decodificar. Em vez de recalcular frequências (e arriscar uma árvore diferente), gravamos a árvore no início do arquivo, em pré-ordem — um bit por nó:

// A árvore vai no cabeçalho, em pré-ordem: '1' + byte para folha, '0' para
// nó interno (e então recursão). É assim que a descompressão a reconstrói.
static bool write_tree(BitWriter *w, const HuffNode *node) {
    if (node->left == NULL && node->right == NULL) {
        return bw_put_bit(w, 1) && bw_put_byte(w, node->byte);
    }
    return bw_put_bit(w, 0) && write_tree(w, node->left) &&
           write_tree(w, node->right);
}

static HuffNode *read_tree(BitReader *r) {
    int tag = br_get_bit(r);
    if (tag == 1) {
        int byte = br_get_byte(r);
        return (byte < 0) ? NULL : leaf_new((unsigned char)byte, 0);
    }
    if (tag == 0) {
        HuffNode *left = read_tree(r);
        HuffNode *right = read_tree(r);
        if (left == NULL || right == NULL) {
            free_tree(left);
            free_tree(right);
            return NULL;
        }
        return internal_new(left, right);
    }
    return NULL; // bits acabaram
}

Com isso, comprimir é: montar a árvore, gerar os códigos, escrever a árvore no cabeçalho e então escrever o código de cada byte do original, bit a bit:

Compressed huff_compress(const unsigned char *data, size_t n) {
    Compressed result = {0};
    result.original_len = n;

    size_t freq[256];
    count_freq(data, n, freq);
    HuffNode *root = build_tree(freq);
    if (root == NULL) {
        return result; // entrada vazia: comprimido vazio
    }

    Code table[256] = {0};
    gen_codes(root, 0, 0, table);

    BitWriter w = {0};
    write_tree(&w, root);            // (cabeçalho) a árvore
    for (size_t i = 0; i < n; i++) { // (payload) o código de cada byte
        Code code = table[data[i]];
        for (int b = code.len - 1; b >= 0; b--) {
            bw_put_bit(&w, (int)((code.bits >> b) & 1u));
        }
    }
    bw_flush(&w);
    free_tree(root);

    result.bytes = w.bytes;
    result.size = w.size;
    return result;
}

E descomprimir é o caminho de volta: reconstruir a árvore do cabeçalho e, para cada byte do original, descer a árvore seguindo os bits lidos até cair numa folha — que revela o byte:

unsigned char *huff_decompress(const Compressed *c, size_t *out_len) {
    if (out_len != NULL) {
        *out_len = c->original_len;
    }
    if (c->original_len == 0) {
        return calloc(1, 1); // vazio, mas não-nulo
    }

    BitReader r = {c->bytes, c->size, 0, 0};
    HuffNode *root = read_tree(&r); // reconstrói a árvore do cabeçalho
    if (root == NULL) {
        return NULL;
    }

    unsigned char *out = malloc(c->original_len);
    if (out == NULL) {
        free_tree(root);
        return NULL;
    }
    for (size_t i = 0; i < c->original_len; i++) {
        const HuffNode *node = root;
        while (node->left != NULL || node->right != NULL) { // desce até a folha
            int bit = br_get_bit(&r);
            if (bit < 0) {
                free(out);
                free_tree(root);
                return NULL;
            }
            node = (bit == 0) ? node->left : node->right;
        }
        out[i] = node->byte;
    }
    free_tree(root);
    return out;
}

11.8 Vendo funcionar

O programa comprime um parágrafo, descomprime de volta e confere se o resultado é idêntico ao original (o teste de round-trip, que prova que não perdemos nada):

int main(void) {
    const char *text =
        "a compressao de huffman da codigos curtos aos simbolos mais "
        "frequentes e codigos longos aos raros. quanto mais repetitivo "
        "o texto, melhor a taxa de compressao. letras como a, e, o e os "
        "espacos aparecem muito; outras, como z, q ou k, quase nunca. e "
        "por isso que vale a pena dar a cada uma um codigo de tamanho "
        "diferente, em vez de gastar 8 bits fixos para todas.";
    size_t n = strlen(text);

    // Comprime e, em seguida, descomprime de volta.
    Compressed c = huff_compress((const unsigned char *)text, n);

    size_t out_len = 0;
    unsigned char *back = huff_decompress(&c, &out_len);
    bool ok = (back != NULL && out_len == n && memcmp(back, text, n) == 0);

    printf("original:   %zu bytes\n", n);
    printf("comprimido: %zu bytes (inclui a arvore no cabecalho)\n", c.size);
    printf("taxa:       %.1f%% do original\n",
           100.0 * (double)c.size / (double)n);
    printf("round-trip: %s\n", ok ? "OK (identico ao original)" : "FALHOU");

    free(back);
    huff_free_compressed(&c);
    return 0;
}

Saída do programa:

original:   361 bytes
comprimido: 221 bytes (inclui a arvore no cabecalho)
taxa:       61.2% do original
round-trip: OK (identico ao original)
DicaComplexidade (Big-O)

Para \(n\) bytes e \(k\) símbolos distintos (\(k \le 256\)): o histograma é \(O(n)\); montar a árvore com o heap é \(O(k \log k)\); gerar os códigos é \(O(k)\); e codificar é \(O(n)\). No total, \(O(n + k \log k)\) — linear no tamanho do arquivo, já que \(k\) é limitado a 256. A taxa de 61% acima é modesta porque o texto é curto e o cabeçalho da árvore pesa; quanto maior e mais repetitivo o arquivo, melhor a taxa — o cabeçalho se dilui e a estatística fica mais clara.

11.9 Veja acontecer

Digite e veja os códigos. Escreva um texto e observe Huffman dar códigos curtos aos caracteres frequentes e longos aos raros — e quanto isso economiza sobre os 8 bits fixos por caractere (só o payload, sem o cabeçalho):

viewof txt = Inputs.text({ value: "abracadabra", label: "Texto", width: 300 })
{
  const s = txt.length ? txt : "abracadabra";
  const freq = new Map();
  for (const ch of s) freq.set(ch, (freq.get(ch) || 0) + 1);

  let nodes = [...freq.entries()].map(([ch, f]) => ({ ch, f, l: null, r: null }));
  if (nodes.length === 1) nodes.push({ ch: null, f: 0, l: null, r: null });
  while (nodes.length > 1) {
    nodes.sort((a, b) => a.f - b.f);
    const a = nodes.shift(), b = nodes.shift();
    nodes.push({ ch: null, f: a.f + b.f, l: a, r: b });
  }
  const codes = new Map();
  (function walk(node, code) {
    if (node.l === null && node.r === null) { codes.set(node.ch, code || "0"); return; }
    walk(node.l, code + "0");
    walk(node.r, code + "1");
  })(nodes[0], "");

  const rows = [...freq.entries()].sort((a, b) => b[1] - a[1]).map(([ch, f]) => {
    const disp = ch === " " ? "␣" : ch;
    const code = codes.get(ch);
    return `<tr><td style="padding:2px 10px"><b>${disp}</b></td><td style="padding:2px 10px">${f}×</td>
      <td style="padding:2px 10px;font-family:monospace;color:#F5793B">${code}</td>
      <td style="padding:2px 10px">${code.length} bit${code.length > 1 ? "s" : ""}</td></tr>`;
  }).join("");

  const huff = [...freq.entries()].reduce((acc, [ch, f]) => acc + f * codes.get(ch).length, 0);
  const fixed = s.length * 8;
  return html`<div>
    <table style="font:13px sans-serif;border-collapse:collapse">
      <tr style="color:#2A9D8F;text-align:left"><th style="padding:2px 10px">símbolo</th><th style="padding:2px 10px">freq.</th><th style="padding:2px 10px">código</th><th style="padding:2px 10px">tam.</th></tr>
      ${rows}
    </table>
    <div style="margin-top:8px;font:13px sans-serif;color:#264653">
      payload: <b style="color:#F5793B">${huff} bits</b> (Huffman) vs <b>${fixed} bits</b> (8 fixos) =
      <b>${(100 * huff / fixed).toFixed(0)}%</b></div>
  </div>`;
}

11.10 Pense como projetista

DicaPense como projetista

Huffman brilha quando há frequências desiguais e você pode pagar uma passada para contá-las. Mas conheça os limites: ele dá códigos de bits inteiros, então um símbolo com probabilidade 0,9 “deveria” custar 0,15 bit e paga 1 — desperdício que a codificação aritmética elimina. Ele também não capta repetições de sequências (“abc” repetido mil vezes); para isso, combina-se Huffman com um estágio de dicionário (LZ77), e é o que os compressores reais fazem. Por fim, guarde a lição estrutural: um problema grande se resolve compondo estruturas simples — array, heap, árvore, bits — cada uma fazendo bem a sua parte.

NotaNo mundo real

Huffman está em todo lugar, quase sempre como o estágio final de algo maior. O DEFLATE (do gzip, do zip e do PNG) combina LZ77 — que substitui repetições por referências — com Huffman. O JPEG aplica Huffman depois da DCT e da quantização para comprimir imagens. Compressores modernos como o Zstandard (zstd) e o Brotli evoluem essa receita com dicionários e modelagem melhores. E a codificação aritmética (e sua prima ANS) é a alternativa que supera o limite de “bits inteiros” do Huffman, ao custo de mais computação.

11.11 Fim da travessia

Você construiu um compactador de arquivos do zero — e, com ele, fechou a Parte da prática. Olhe para trás e veja quanta coisa se encaixou neste último projeto:

  • o array para o histograma e para guardar o heap (cap. 5);
  • a noção de nó e ponteiro das listas, agora em forma de árvore (cap. 6);
  • a generalidade do void* na disciplina de tratar bytes crus (cap. 7);
  • a fila (FIFO) reaproveitada como ideia, e a fila de prioridade como peça central (caps. 8 e 10);
  • a árvore e seus percursos recursivos (cap. 9);
  • e, sob tudo, os ponteiros, a memória e os bits dos fundamentos.

Nenhuma dessas estruturas é difícil sozinha. A maestria do projetista está em escolher a certa para cada parte e compô-las num todo que funciona — e isso, agora, você fez. As próximas páginas (a Parte de fundamentos) destrincham o C que sustentou tudo isto; leia-as quando quiser firmar o chão sob os pés.
