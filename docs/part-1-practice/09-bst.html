9  Árvores de busca: organizando para achar rápido

Todas as estruturas até aqui buscam em \(O(n)\): array, lista, pilha, fila — para achar um valor, no pior caso, é preciso olhar todos. Neste capítulo encontramos a primeira estrutura em que buscar pode custar \(O(\log n)\): a árvore binária de busca (BST). O truque não é mágica — é organização. Em vez de uma sequência plana, os dados ficam numa hierarquia com uma regra que permite, a cada passo, descartar metade do que sobrou.

9.1 A analogia: adivinhe o número

No jogo de adivinhar um número de 1 a 100, a melhor estratégia é chutar 50: a resposta “maior” ou “menor” elimina metade das possibilidades. Cada palpite seguinte corta a metade que sobrou. Em ~7 tentativas você acha qualquer número — porque \(\log_2 100 \approx 7\).

A árvore binária de busca materializa essa estratégia. Cada nó é um “palpite”; ir para a esquerda ou para a direita é o “menor/maior”. Se a árvore estiver equilibrada, cada passo descarta metade dos nós, e a busca custa \(O(\log n)\).

9.2 A invariante de ordenação

A BST é uma árvore binária (cada nó tem até dois filhos) com uma regra de ouro:

Para todo nó, tudo na subárvore esquerda é menor, e tudo na subárvore direita é maior.

Essa invariante, mantida em toda inserção, é o que permite buscar descartando metades. O nó carrega o valor e dois ponteiros:

/*
 * bst.h — Árvore Binária de Busca (BST) de inteiros.
 *
 * Projeto:   PIESC — Estruturas de Dados e Algoritmos em C
 * Capítulo:  9 — Árvores Binárias de Busca
 * Padrão:    C17 (ISO/IEC 9899:2018)
 * Build:     gcc/clang -std=c17 -Wall -Wextra -Wpedantic   (ver Makefile)
 */
#ifndef BST_H
#define BST_H

#include <stddef.h>  // size_t
#include <stdbool.h> // bool

typedef struct BSTNode {
    int value;
    struct BSTNode *left;  // subárvore com valores MENORES
    struct BSTNode *right; // subárvore com valores MAIORES
} BSTNode;

typedef struct {
    BSTNode *root;
    size_t size;
} BST;

// Função aplicada a cada valor durante um percurso.
typedef void (*VisitFn)(int value, void *ctx);

void bst_init(BST *t);
void bst_free(BST *t);

bool bst_insert(BST *t, int value); // ignora duplicatas
bool bst_contains(const BST *t, int value);
bool bst_remove(BST *t, int value); // trata os três casos
size_t bst_height(const BST *t);    // árvore vazia = 0
size_t bst_size(const BST *t);

void bst_preorder(const BST *t, VisitFn visit, void *ctx);
void bst_inorder(const BST *t, VisitFn visit, void *ctx); // sai ORDENADO
void bst_postorder(const BST *t, VisitFn visit, void *ctx);
void bst_levelorder(const BST *t, VisitFn visit, void *ctx); // BFS (usa fila)

#endif // BST_H

A Figura 9.1 mostra a árvore que usaremos no capítulo. Confira a invariante em qualquer nó: à esquerda do 50 estão 30, 20, 40 (todos menores); à direita, 70, 60, 80 (todos maiores).

9.3 Inserção: descer até achar o lugar

Inserir é seguir a invariante até cair num ponto vazio (NULL) e criar ali uma folha. A cada nó, vamos para a esquerda (se menor) ou direita (se maior):

static BSTNode *node_new(int value) {
    BSTNode *n = malloc(sizeof(BSTNode));
    if (n != NULL) {
        n->value = value;
        n->left = NULL;
        n->right = NULL;
    }
    return n;
}

// Desce recursivamente: menor vai à esquerda, maior à direita. O(altura).
static BSTNode *insert_rec(BSTNode *node, int value, bool *ok) {
    if (node == NULL) {
        BSTNode *fresh = node_new(value); // achou o lugar: cria a folha
        *ok = (fresh != NULL);
        return (fresh != NULL) ? fresh : node;
    }
    if (value < node->value) {
        node->left = insert_rec(node->left, value, ok);
    } else if (value > node->value) {
        node->right = insert_rec(node->right, value, ok);
    } else {
        *ok = false; // valor já existe: ignora a duplicata
    }
    return node;
}

bool bst_insert(BST *t, int value) {
    bool ok = false;
    t->root = insert_rec(t->root, value, &ok);
    if (ok) {
        t->size++;
    }
    return ok;
}

O custo é \(O(h)\), onde \(h\) é a altura da árvore — quantos níveis descemos. Guarde esse “\(O(h)\)”; toda a riqueza (e o perigo) da BST está em quanto vale \(h\).

9.4 Busca: a cada passo, descarte metade

Buscar é o mesmo movimento, sem criar nada: compare, vá para um lado, repita.

// Busca iterativa: a cada nó, vai para um lado só. O(altura).
bool bst_contains(const BST *t, int value) {
    const BSTNode *cur = t->root;
    while (cur != NULL) {
        if (value == cur->value) {
            return true;
        }
        cur = (value < cur->value) ? cur->left : cur->right;
    }
    return false;
}

Numa árvore equilibrada, cada comparação elimina metade dos nós restantes — é o jogo da adivinhação. Por isso a busca é \(O(h) = O(\log n)\) no caso equilibrado. Compare com a lista do capítulo 6, onde achar custa sempre \(O(n)\): a árvore não varre, ela navega.

9.5 Percursos: visitar todos os nós

Às vezes queremos percorrer a árvore inteira. Há quatro ordens clássicas. As três em profundidade (DFS) são recursivas e diferem apenas na hora de visitar a raiz em relação aos filhos:

// Os três percursos em profundidade diferem APENAS na posição do visit():
static void preorder_rec(const BSTNode *n, VisitFn visit, void *ctx) {
    if (n == NULL)
        return;
    visit(n->value, ctx); // raiz primeiro
    preorder_rec(n->left, visit, ctx);
    preorder_rec(n->right, visit, ctx);
}
static void inorder_rec(const BSTNode *n, VisitFn visit, void *ctx) {
    if (n == NULL)
        return;
    inorder_rec(n->left, visit, ctx);
    visit(n->value, ctx); // raiz no meio → sai em ordem CRESCENTE
    inorder_rec(n->right, visit, ctx);
}
static void postorder_rec(const BSTNode *n, VisitFn visit, void *ctx) {
    if (n == NULL)
        return;
    postorder_rec(n->left, visit, ctx);
    postorder_rec(n->right, visit, ctx);
    visit(n->value, ctx); // raiz por último
}
DicaIdeia central

O percurso em-ordem (esquerda → raiz → direita) visita os valores em ordem crescente — de graça. É consequência direta da invariante: tudo à esquerda é menor, tudo à direita é maior. Ou seja: uma BST é, implicitamente, seus dados ordenados. Foi exatamente o que a saída 20 30 40 50 60 70 80 mostrou.

A quarta ordem é em largura (BFS): visitar nível por nível, de cima para baixo. Ela não é recursiva — precisa de uma fila para lembrar os próximos nós a visitar, reaproveitando a ideia FIFO do capítulo 8:

// Percurso em largura (BFS): visita nível a nível, usando uma FILA de
// ponteiros — a mesma ideia FIFO do capítulo 8.
void bst_levelorder(const BST *t, VisitFn visit, void *ctx) {
    if (t->root == NULL) {
        return;
    }
    const BSTNode **queue = malloc(t->size * sizeof(*queue));
    if (queue == NULL) {
        return;
    }
    size_t head = 0, tail = 0;
    queue[tail++] = t->root; // enfileira a raiz
    while (head < tail) {
        const BSTNode *node = queue[head++]; // desenfileira a frente
        visit(node->value, ctx);
        if (node->left != NULL) {
            queue[tail++] = node->left; // enfileira os filhos
        }
        if (node->right != NULL) {
            queue[tail++] = node->right;
        }
    }
    free(queue);
}

9.6 Remoção: os três casos

Remover é a operação mais delicada, porque não podemos deixar um buraco que quebre a invariante. Há três casos, em dificuldade crescente:

  1. Folha (sem filhos): basta liberar o nó e apontar o pai para NULL.
  2. Um filho: o pai “adota” o único neto — o filho sobe no lugar do removido.
  3. Dois filhos: o caso difícil. Não dá para escolher um filho para subir sem desorganizar tudo. A solução: achar o sucessor in-order (o menor valor da subárvore direita), copiar seu valor para o nó que íamos remover, e então remover o sucessor lá embaixo — que, por ser o menor, nunca tem filho à esquerda, recaindo no caso 1 ou 2.
// O menor valor de uma subárvore é o nó mais à esquerda.
static BSTNode *find_min(BSTNode *node) {
    while (node->left != NULL) {
        node = node->left;
    }
    return node;
}

static BSTNode *remove_rec(BSTNode *node, int value, bool *removed) {
    if (node == NULL) {
        return NULL; // não encontrado
    }
    if (value < node->value) {
        node->left = remove_rec(node->left, value, removed);
    } else if (value > node->value) {
        node->right = remove_rec(node->right, value, removed);
    } else {
        *removed = true;
        // Caso 1 e 2: nó sem filho à esquerda — devolve o filho à direita
        // (que pode ser NULL). Cobre folha e nó com um único filho.
        if (node->left == NULL) {
            BSTNode *right = node->right;
            free(node);
            return right;
        }
        if (node->right == NULL) {
            BSTNode *left = node->left;
            free(node);
            return left;
        }
        // Caso 3: dois filhos. Copia o SUCESSOR in-order (o menor da
        // subárvore direita) para cá e remove o sucessor lá embaixo.
        BSTNode *succ = find_min(node->right);
        node->value = succ->value;
        node->right = remove_rec(node->right, succ->value, removed);
    }
    return node;
}

bool bst_remove(BST *t, int value) {
    bool removed = false;
    t->root = remove_rec(t->root, value, &removed);
    if (removed) {
        t->size--;
    }
    return removed;
}

No demo, removemos a raiz 50 (que tem dois filhos): o sucessor 60 assume seu lugar, e o inorder continua ordenado — 20 30 40 60 70 80. A invariante sobreviveu.

9.7 Vendo funcionar

O programa monta a árvore da Figura 9.1, imprime os quatro percursos, busca dois valores e remove a raiz:

static void print_int(int value, void *ctx) {
    (void)ctx;
    printf(" %d", value);
}

int main(void) {
    BST t;
    bst_init(&t);

    // Inserções formam a árvore equilibrada usada no capítulo.
    int vals[] = {50, 30, 70, 20, 40, 60, 80};
    for (size_t i = 0; i < sizeof(vals) / sizeof(vals[0]); i++) {
        bst_insert(&t, vals[i]);
    }

    printf("inorder (ordenado):");
    bst_inorder(&t, print_int, NULL);
    printf("\npreorder          :");
    bst_preorder(&t, print_int, NULL);
    printf("\npostorder         :");
    bst_postorder(&t, print_int, NULL);
    printf("\nlevelorder (BFS)  :");
    bst_levelorder(&t, print_int, NULL);

    printf("\ncontem 40? %s | contem 45? %s\n",
           bst_contains(&t, 40) ? "sim" : "nao",
           bst_contains(&t, 45) ? "sim" : "nao");
    printf("altura: %zu\n", bst_height(&t));

    // Remover a raiz (50): tem dois filhos, então o sucessor (60) assume.
    bst_remove(&t, 50);
    printf("apos remover 50    :");
    bst_inorder(&t, print_int, NULL);
    printf("\n");

    bst_free(&t);
    return 0;
}

postorder         : 20 40 30 60 80 70 50
levelorder (BFS)  : 50 30 70 20 40 60 80

altura: 3
apos remover 50    : 20 30 40 60 70 80

9.8 A altura é tudo: balanceamento e degeneração

Repetimos que as operações são \(O(h)\). Quando \(h\) vale \(O(\log n)\), a BST é maravilhosa. Mas \(h\) depende de como os dados entraram. Insira 10, 20, 30, 40, 50 em ordem crescente e veja o desastre da Figura 9.2: cada nó só tem filho à direita. A árvore degenera numa lista — altura \(n\), e toda operação volta a ser \(O(n)\).

static size_t height_rec(const BSTNode *node) {
    if (node == NULL) {
        return 0;
    }
    size_t l = height_rec(node->left);
    size_t r = height_rec(node->right);
    return 1 + (l > r ? l : r);
}

size_t bst_height(const BST *t) {
    return height_rec(t->root);
}
ImportanteTrade-off: Big-O médio × pior caso

A BST tem dois rostos:

  • Caso médio (inserções em ordem aleatória): altura \(O(\log n)\) — busca, inserção e remoção em \(O(\log n)\). Rápida e elegante.
  • Pior caso (inserções já ordenadas, ou quase): altura \(O(n)\) — tudo degenera para \(O(n)\), pior que um array para muitas tarefas.

Essa fragilidade é a maior limitação da BST simples. A solução não é abandoná-la, e sim rebalanceá-la automaticamente a cada inserção/remoção — é o que fazem as árvores balanceadas do box “No mundo real”.

9.9 Veja acontecer

Os percursos lado a lado. Escolha uma ordem e veja, sobre a mesma árvore, em que sequência os nós são visitados (o número verde é a ordem de visita):

viewof trav = Inputs.radio(["pré-ordem", "em-ordem", "pós-ordem", "em-largura (BFS)"], { value: "em-ordem", label: "Percurso" })
bst_nodes = ({
  50: { x: 150, y: 28 }, 30: { x: 82, y: 92 }, 70: { x: 218, y: 92 },
  20: { x: 40, y: 156 }, 40: { x: 124, y: 156 }, 60: { x: 176, y: 156 }, 80: { x: 260, y: 156 }
})
bst_edges = [[50, 30], [50, 70], [30, 20], [30, 40], [70, 60], [70, 80]]
bst_orders = ({
  "pré-ordem": [50, 30, 20, 40, 70, 60, 80],
  "em-ordem": [20, 30, 40, 50, 60, 70, 80],
  "pós-ordem": [20, 40, 30, 60, 80, 70, 50],
  "em-largura (BFS)": [50, 30, 70, 20, 40, 60, 80]
})
{
  const order = bst_orders[trav];
  const idx = new Map(order.map((v, i) => [v, i + 1]));
  const edges = bst_edges.map(([a, b]) => {
    const pa = bst_nodes[a], pb = bst_nodes[b];
    return `<line x1="${pa.x}" y1="${pa.y}" x2="${pb.x}" y2="${pb.y}" stroke="#264653" stroke-width="1.5"/>`;
  }).join("");
  const nodes = Object.entries(bst_nodes).map(([v, p]) => `
    <circle cx="${p.x}" cy="${p.y}" r="16" fill="#F5793B" stroke="#fff" stroke-width="2"/>
    <text x="${p.x}" y="${p.y + 4}" text-anchor="middle" fill="#fff" font-size="12" font-weight="700">${v}</text>
    <circle cx="${p.x + 15}" cy="${p.y - 15}" r="9" fill="#2A9D8F"/>
    <text x="${p.x + 15}" y="${p.y - 11}" text-anchor="middle" fill="#fff" font-size="10" font-weight="700">${idx.get(+v)}</text>`).join("");
  return html`<div>
    <svg viewBox="0 0 300 185" style="max-width:340px;width:100%">${edges}${nodes}</svg>
    <div style="font:14px sans-serif;color:#264653;margin-top:4px">ordem de visita: <b>${order.join(" → ")}</b></div>
  </div>`;
}

9.10 Pense como projetista

DicaPense como projetista

Use uma árvore de busca quando precisar manter um conjunto ordenado com inserção, remoção e busca todas eficientes, ou quando consultas por faixa (“todos entre 30 e 60”) e “próximo maior/menor” importam — coisas que uma tabela hash não dá. Mas não escreva uma BST simples para produção: a degeneração para \(O(n)\) é fácil de disparar (dados quase ordenados são comuns). Use uma árvore balanceada pronta (ver abaixo) ou a estrutura ordenada da sua linguagem. A BST crua é para entender o mecanismo — que é exatamente o que estamos fazendo.

NotaNo mundo real: árvores balanceadas

A resposta à degeneração são as árvores auto-balanceadas, que fazem pequenas rotações a cada inserção/remoção para manter a altura em \(O(\log n)\) garantido. As AVL (mais rígidas, busca mais rápida) e as Red-Black (rebalanceiam menos, escrita mais rápida) são as clássicas: a Red-Black está por trás do std::map/std::set do C++, do TreeMap do Java e de várias estruturas do kernel do Linux (escalonador, gerência de memória). É a BST deste capítulo, blindada contra o pior caso.

NotaNo mundo real: árvores de disco e de texto

Quando os dados não cabem na RAM, entram as B-trees e B+-trees: árvores “gordas” com centenas de filhos por nó, desenhadas para minimizar leituras de disco/SSD. São a espinha dorsal dos índices de bancos de dados (PostgreSQL, MySQL) e de sistemas de arquivos (NTFS, ext4). Para texto, as tries (árvores de prefixo) e radix trees indexam por caractere e brilham em autocompletar e roteamento de IP.

9.11 Resumo

  • A BST mantém a invariante de ordenação (esquerda menor, direita maior), que permite buscar descartando metade a cada passo — como o jogo de adivinhar.
  • Inserção, busca e remoção são todas \(O(h)\), onde \(h\) é a altura.
  • O percurso em-ordem entrega os dados ordenados; o BFS visita por níveis usando uma fila (cap. 8).
  • A remoção tem três casos; o difícil (dois filhos) se resolve com o sucessor in-order.
  • O calcanhar de Aquiles é a altura: dados ordenados degeneram a BST numa lista (\(O(n)\)). A cura são as árvores balanceadas (AVL, Red-Black).
